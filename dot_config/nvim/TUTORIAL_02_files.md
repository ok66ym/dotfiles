# チュートリアル 2: ファイル操作

neo-tree・Telescope・バッファ・ウィンドウ分割を身につける。

`<Leader>` = `Space`  |  インデックス: `<Leader>?r`  |  前: `<Leader>?1`  |  次: `<Leader>?3`

---

## Step 1: nvim . でフォルダを開く

```
nvim .
```

実行すると neo-tree（ファイルエクスプローラー）が左側に自動で開く。

### neo-tree 内の操作

| キー | 動作 |
|------|------|
| `j / k` | ファイル・ディレクトリを選択 |
| `Enter` または `l` | ファイルを開く / ディレクトリを展開 |
| `h` | ディレクトリを折りたたむ |
| `a` | ファイル・ディレクトリを新規作成（末尾に `/` でディレクトリ） |
| `d` | 削除（確認あり） |
| `r` | リネーム |
| `y` | ファイルパスをコピー |
| `q` | neo-tree を閉じる |
| `?` | neo-tree のキーバインド一覧 |

### ファイルを開いた後の流れ

ファイルを選択して `Enter` を押すと neo-tree が自動で閉じ、エディタ領域でファイルが開く。

### neo-tree とエディタ間のフォーカス移動

| キー | 動作 |
|------|------|
| `<Leader>e` | neo-tree を開いてフォーカスを移す（既に開いていればフォーカスのみ移動） |
| `Ctrl+l` | neo-tree からエディタ側のペインへ移動 |
| `q` | neo-tree を閉じる |

```
エディタ → neo-tree : <Leader>e
neo-tree → エディタ : Ctrl+l（または Enter でファイルを開く）
```

**neo-tree にフォーカスがある時は `<Leader>` キーが効かない。**
`<Leader>?` などでドキュメントを開く場合は `Ctrl+l` でエディタ側へ移動してから操作する。

### フィルター検索とキャンセル

neo-tree 内で `/` を押すとファイル名のフィルタリングができる。

| キー | 動作 |
|------|------|
| `/` | フィルター入力開始 |
| `Esc` | フィルターをキャンセルして全ファイル表示に戻る |
| `<` | フィルタークリア |

### 練習 1-A

1. `nvim .` で起動 → neo-tree が開いた状態を確認
2. `j/k` でファイルを選択
3. `l` でディレクトリを展開 / `h` で折りたたむ
4. `Enter` でファイルを開く → neo-tree が自動で閉じることを確認
5. `<Leader>e` で neo-tree を再度開く
6. `q` で neo-tree を閉じる

## Step 2: Telescope でファイルを検索して開く（最もよく使う）

`nvim .` や neo-tree を使わなくても Telescope で直接ファイルを開ける。

```
<Leader>ff  → ファイル名検索
```

| キー | 動作 |
|------|------|
| `<Leader>ff` | プロジェクト内のファイルを名前で検索 |
| `<Leader>fr` | 最近開いたファイル一覧 |
| `<Leader>fg` | プロジェクト全体をキーワードで全文検索 |
| `<Leader>fb` | 現在開いているバッファ一覧 |

Telescope 内の操作:

| キー | 動作 |
|------|------|
| 文字を入力 | リアルタイムで絞り込み |
| `Ctrl+j / Ctrl+k` | リストを上下移動 |
| `Enter` | 選択したファイルを開く |
| `Ctrl+v` | 垂直分割で開く |
| `Ctrl+x` | 水平分割で開く |
| `Esc` | Telescope を閉じる |

### 練習 2-A

1. `<Leader>ff` でファイル検索を開く
2. ファイル名の一部を入力して絞り込む
3. `Ctrl+j` でリストを下に移動
4. `Enter` でファイルを開く
5. `<Leader>fg` で全文検索を試す（コード内のキーワードで検索）

## Step 3: バッファ管理（複数ファイルの切り替え・閉じる）

### 概念の整理

| 用語 | 意味 |
|------|------|
| バッファ | 開いているファイルのデータ（メモリ上）。上部タブバーに表示される |
| ウィンドウ | バッファを表示する画面の分割領域 |

### バッファの切り替え

| キー | 動作 |
|------|------|
| `Shift+h` | 前のバッファへ |
| `Shift+l` | 次のバッファへ |
| `<Leader>fb` | バッファ一覧から選択（Telescope） |

### バッファを閉じる

| キー / コマンド | 動作 |
|----------------|------|
| `<Leader>bd` | 現在のバッファを閉じる（タブバーから消える） |
| `:q` | ウィンドウを閉じる（バッファはタブバーに残る） |
| `:bd` | バッファを削除（タブバーから消える） |
| `:wq` / `ZZ` | 保存して閉じる |
| `:q!` / `ZQ` | 保存せずに閉じる |

> `:q` → ウィンドウだけ閉じる（バッファはタブバーに残る）
> `:bd` → バッファも削除する（タブバーから消える）

### 練習 3-A

1. `<Leader>ff` で複数のファイルを順番に開く（3〜4ファイル）
2. 上部のタブバーに複数のバッファが表示されることを確認
3. `Shift+l` / `Shift+h` でバッファを切り替える
4. `<Leader>fb` でバッファ一覧から目的のファイルを選択
5. `<Leader>bd` でバッファを閉じる → タブバーから消えることを確認
6. `:q` で閉じる → タブバーに残ることを確認（挙動の違いを体感）

## Step 4: ウィンドウを分割して複数ファイルを並べる

1つの Neovim 画面内に複数のファイルを並べて表示できる。

### 方法 A: Telescope から分割で開く（推奨・最も簡単）

`<Leader>ff` でファイルを選択中に:

| キー | 動作 |
|------|------|
| `Enter` | 現在のウィンドウで開く |
| `Ctrl+v` | 右に垂直分割して開く |
| `Ctrl+x` | 下に水平分割して開く |

### 方法 B: コマンドで分割する

| コマンド | 動作 |
|---------|------|
| `:vsp path/to/file` | 別ファイルを右に垂直分割で開く |
| `:sp path/to/file` | 別ファイルを下に水平分割で開く |
| `:vsplit` / `:split` | 現在のファイルを複製して分割 |

### 方法 C: キーバインドで分割してから別ファイルを開く

```
1. Ctrl+w v で垂直分割（右に同じファイル）
2. Ctrl+l で右ウィンドウに移動
3. <Leader>ff で別のファイルを開く
```

### ウィンドウ間の移動・操作

| キー | 動作 |
|------|------|
| `Ctrl+h` | 左のウィンドウへ移動 |
| `Ctrl+j` | 下のウィンドウへ移動 |
| `Ctrl+k` | 上のウィンドウへ移動 |
| `Ctrl+l` | 右のウィンドウへ移動 |
| `<Leader>w=` | 全ウィンドウを均等サイズに |
| `:q` | 現在のウィンドウを閉じる |

> `Ctrl+h/j/k/l` は Neovim のウィンドウ移動と tmux のペイン移動が統合されている。
> Neovim 外の tmux ペインにも同じキーで移動できる。

### 分割したウィンドウで別のファイルを開く

```
1. Ctrl+w v で分割する
2. Ctrl+l で右ウィンドウへ移動
3. <Leader>ff で別のファイルを開く
```

### 練習 4-A

1. ファイルを1つ開く
2. `Ctrl+w v` で垂直分割
3. `Ctrl+l` で右ウィンドウへ移動
4. `<Leader>ff` で別のファイルを開く
5. `Ctrl+h` で左ウィンドウへ戻る
6. `:q` で右ウィンドウを閉じる

## Step 5: エクスプローラーの操作まとめ

| キー | 動作 |
|------|------|
| `<Leader>e` | neo-tree を開いてフォーカスを移す（閉じていれば開く） |
| `<Leader>o` | 現在ファイルをツリー内でハイライト表示してフォーカスを移す |
| `q`（neo-tree 内） | neo-tree を閉じてエディタに戻る |
| `Ctrl+l`（neo-tree 内） | エディタ側にフォーカスを移す（neo-tree は開いたまま） |
