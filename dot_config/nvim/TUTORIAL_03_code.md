# チュートリアル 3: コード操作

検索・置換・LSP・補完・Git を身につける。

`<Leader>` = `Space`  |  インデックス: `<Leader>?r`  |  前: `<Leader>?2`  |  次: `<Leader>?4`

## Step 1: ファイル内の検索・置換

### 検索

| キー | 動作 |
|------|------|
| `/{検索語}` + `Enter` | 下方向に検索 |
| `?{検索語}` + `Enter` | 上方向に検索 |
| `n` | 次の一致箇所へ |
| `N` | 前の一致箇所へ |
| `*` | カーソル下の単語を検索 |
| `Esc` | 検索ハイライトを消す |

### 置換（Command モード）

```
:%s/old/new/g     → ファイル全体で置換
:%s/old/new/gc    → 確認しながら置換（y=実行 / n=スキップ / q=中止）
:10,20s/old/new/g → 10〜20行目の範囲で置換
```

### フロートウィンドウ内でも検索できる

`<Leader>?u` などで開いたドキュメントのフロートウィンドウ内でも `/` で検索できる。

```
/Step   → 「Step」を検索
n       → 次の「Step」へ
N       → 前の「Step」へ
Esc     → 検索を終了
```

### 練習 1-A

1. ファイルを開いた状態で `/function` + `Enter`
2. `n` で次の一致箇所へ移動
3. `*` でカーソル下の単語を検索
4. `Esc` でハイライトを消す

## Step 2: プロジェクト全体の検索（Telescope）

```
<Leader>fg  → プロジェクト全体をキーワードで検索（live grep）
```

入力しながらリアルタイムで絞り込まれる。`Enter` でその行を開く。

| キー | 動作 |
|------|------|
| `<Leader>fg` | プロジェクト全体検索 |
| `<Leader>/` | 現在ファイル内のファジー検索 |
| `<Leader>fs` | LSP シンボル検索（関数名・クラス名） |

### 練習 2-A

1. `<Leader>fg` でプロジェクト全体検索を開く
2. 関数名やクラス名を入力して絞り込む
3. `Ctrl+j` でリストを移動して `Enter` でそのファイル・行を開く

## Step 3: LSP 補完

Insert モードで文字を入力すると補完候補が自動表示される。

| キー | 動作 |
|------|------|
| `Ctrl+j / Ctrl+k` | 候補を上下移動 |
| `Tab` | 候補を選択 / スニペットの次フィールドへ |
| `Shift+Tab` | 逆方向に移動 |
| `Enter` | 候補を確定 |
| `Ctrl+Space` | 補完を強制表示 |
| `Ctrl+e` | 補完ウィンドウを閉じる |

### 練習 3-A

1. TypeScript または Ruby ファイルを開く
2. `i` でInsertモードへ
3. 関数名の途中まで入力して補完候補を確認
4. `Tab` で選択して `Enter` で確定

## Step 4: LSP ジャンプ（定義・参照・実装）

コードの定義元・参照先・実装に飛ぶ。

| キー | 動作 | 飛び先 |
|------|------|--------|
| `gd` | definition | 型定義・宣言（.d.ts や import 元） |
| `gi` | implementation | 実際の実装コード |
| `gr` | references | 使われている全箇所（Telescope で一覧表示） |
| `K` | hover | ドキュメント・型情報をポップアップ表示 |
| `Ctrl+o` | 前の場所へ戻る | ジャンプ前の位置に戻る |

### React コンポーネントへのジャンプ

```tsx
<MyComponent />
```

`MyComponent` の上にカーソルを置いて `gd` → コンポーネントの定義ファイルへ飛ぶ。

> バレルファイル（index.ts）経由でexportされている場合:
> 1回目の `gd` → index.ts に飛ぶ
> 2回目の `gd` → 実際の定義ファイルに飛ぶ
>
> import 文の上で `gd` → そのモジュールのファイル先頭に飛ぶ。
> 実装コードに飛びたい場合は `gi` を使う。
> 飛んだ後は `Ctrl+o` で元の場所に戻れる。

### LSP が動いているか確認する

`gd` などが動かない場合はまず LSP が起動しているか確認する。

```
:LspInfo    → 現在のファイルに対して起動している LSP サーバーの一覧
:Mason      → LSP サーバーのインストール状況（✓ = インストール済み）
:LspRestart → LSP サーバーを再起動
```

`:LspInfo` で `ts_ls: no active clients` と表示される場合:

1. `.tsx` / `.ts` ファイルを開いた状態で `:LspInfo` を確認
2. `tsconfig.json` または `package.json` が存在するディレクトリで `nvim .` を開いているか確認
3. `:Mason` で `ts_ls` が `✓` になっているか確認
4. worktree 環境の場合 `node_modules` が存在するか確認（未インストールなら `npm i`）

## Step 5: Git 操作

### 変更を確認する（inline）

ファイルを編集すると行番号の左側（ガター）に変更サインが表示される。

| 表示 | 意味 |
|------|------|
| `│`（緑） | 追加行 |
| `│`（黄） | 変更行 |
| `_` / `‾` | 削除行 |

| キー | 動作 |
|------|------|
| `]h` | 次の変更ハンクへ移動 |
| `[h` | 前の変更ハンクへ移動 |
| `<Leader>gp` | 変更内容のプレビュー（変更前後を表示） |
| `<Leader>gd` | 現在ファイルの diff を分割表示（変更前後を並べて確認） |
| `<Leader>gb` | 行の git blame（誰がいつ書いたか） |

### diffview で差分確認・ステージング

複数ファイルの差分をファイルツリー付きで確認しながらステージングできる。

| キー | 動作 |
|------|------|
| `<Leader>gv` | 差分ビューを開く（全変更ファイル一覧 + diff） |
| `<Leader>gh` | 現在ファイルのコミット履歴 |
| `<Leader>gH` | リポジトリ全体のコミット履歴 |

diffview は「ファイルパネル（左）」と「diff ビュー（右）」の 2 ペインで構成される。

ファイルパネル（左）— ステージング操作:

| キー | 動作 |
|------|------|
| `j / k` | ファイルを上下移動 |
| `Enter` / `l` | 選択ファイルの diff を表示 |
| `s` または `-` | 選択ファイルをステージ / アンステージ |
| `S` | 全ファイルをステージ |
| `U` | 全ファイルをアンステージ |
| `X` | 選択ファイルの変更を破棄 |

diff ビュー（右）— 差分確認:

| キー | 動作 |
|------|------|
| `]c / [c` | 次/前の変更ハンクへジャンプ |
| `<Leader>e` | ファイルパネルにフォーカスを移す |
| `g?` | キーバインド一覧を表示 |
| `q` | diffview を閉じる |

> ステージングを diffview で行い、コミット・Push は lazygit（`<Leader>gg`）で行う流れが効率的。

### lazygit でコミット・Push

```
<Leader>gg  → lazygit を開く
```

lazygit 内のパネル:

| キー | パネル |
|------|--------|
| `1` | Files（変更ファイル一覧） |
| `2` | Branches |
| `3` | Commits |
| `4` | Stash |

lazygit 内の操作:

| キー | 動作 |
|------|------|
| `Space` | ファイルをステージ / アンステージ |
| `d` | 選択ファイルの diff を表示 |
| `c` | コミット（メッセージ入力 → Enter） |
| `P` | Push |
| `p` | Pull |
| `b` | ブランチ操作 |
| `Esc` | 現在のパネル・モードをキャンセル |
| `q` | lazygit を閉じて Neovim に戻る |

> `<Leader>gl` → リポジトリ全体のコミットログ
> `<Leader>gL` → 現在ファイルのコミットログ

### 練習 5-A: gitsigns で inline 確認

1. ファイルを適当に編集して保存（`:w`）
2. ガターに `│` が表示されることを確認
3. `]h` で変更ハンクへ移動
4. `<Leader>gp` で変更前後の差分を確認

### 練習 5-B: diffview でステージング

1. `<Leader>gv` で差分ビューを開く
2. ファイルパネル（左）で `j/k` を使いファイルを確認
3. `l` で選択ファイルの diff を diff ビュー（右）に表示
4. `]c / [c` で diff ビュー内のハンクを移動
5. ファイルパネルに戻り `s` でステージ
6. `S` で残りの全ファイルをステージ
7. `q` で diffview を閉じる

### 練習 5-C: lazygit でコミット

1. `<Leader>gg` で lazygit を開く
2. `Space` でファイルをステージ（diffview でステージ済みの場合は確認のみ）
3. `d` で diff を確認
4. `c` でコミット（メッセージ入力 → Enter）
5. `q` で lazygit を閉じて Neovim に戻る
