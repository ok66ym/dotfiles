# Neovim 操作ガイド

Ghostty + tmux + Claude Code 環境での Neovim 操作まとめ。
プラグインの詳細は `PLUGINS.md` を参照。

`<Leader>` = `Space`

---

## 基本移動

| キー | 動作 |
|------|------|
| `h/j/k/l` | カーソル移動（左/下/上/右） |
| `w/b/e` | 単語単位で移動（次頭/前頭/語尾） |
| `0 / $` | 行頭 / 行末 |
| `gg / G` | ファイル先頭 / 末尾 |
| `Ctrl+f/b` | 1ページ下/上 |
| `Ctrl+d/u` | 半ページ下/上 |
| `Ctrl+o/i` | ジャンプリストを戻る/進む |
| `s + 2文字` | 画面内を高速ジャンプ（Flash） |

## nvim . でフォルダを開いた時の操作

### 起動時の動作

```
nvim .
```

を実行すると neo-tree（ファイルエクスプローラー）が左サイドに自動で開く。
カーソルは neo-tree 内にある状態で始まる。

### neo-tree とエディタのフォーカス切り替え

| キー | 動作 |
|------|------|
| `<Leader>e` | neo-tree にフォーカス（閉じていれば開く / 開いていればフォーカスを移す） |
| `Ctrl+l` | neo-tree → エディタ側のペインへ移動（neo-tree は開いたまま） |
| `q`（neo-tree 内） | neo-tree を閉じる |

> neo-tree にフォーカスがある時は `<Leader>` キーが効かない。
> ドキュメント参照（`<Leader>?u` 等）は `Ctrl+l` でエディタに移動してから行う。

### neo-tree 内の操作

| キー | 動作 |
|------|------|
| `j / k` | ファイル・ディレクトリを上下に選択 |
| `Enter` または `l` | ファイルを開く / ディレクトリを展開 |
| `h` | ディレクトリを折りたたむ |
| `a` | ファイルまたはディレクトリを新規作成（末尾に `/` でディレクトリ） |
| `d` | 削除（確認あり） |
| `r` | リネーム |
| `y` | ファイルパスをコピー |
| `p` | ファイルを貼り付け |
| `q` | neo-tree を閉じる |
| `?` | neo-tree のキーバインド一覧を表示 |

### ファイルを開いた後の流れ

ファイルを選択（`Enter` / `l`）すると neo-tree が自動で閉じ、エディタ領域でファイルが開く。

```
j/k でファイルを選択
  ↓
Enter でファイルを開く → neo-tree が自動で閉じる
  ↓
通常の Neovim 操作で編集（i でインサート、jj で抜ける）
  ↓
:w で保存
```

### エクスプローラーを再表示したい時

| キー | 動作 |
|------|------|
| `<Leader>e` | neo-tree をトグル（開く / 閉じる） |
| `<Leader>o` | 現在編集中のファイルをツリー内でハイライト表示 |

### 別のファイルを追加で開く

neo-tree を経由せずに直接ファイルを開く方法：

| キー | 動作 |
|------|------|
| `<Leader>ff` | ファイル名で検索して開く（Telescope） |
| `<Leader>fr` | 最近開いたファイル一覧から開く（Telescope） |
| `:e path/to/file` | パスを直接指定して開く |

## コード操作（LSP）

| キー | 動作 | 飛び先 |
|------|------|--------|
| `gd` | 定義へジャンプ | 型定義・宣言（.d.ts 等） |
| `gi` | 実装へジャンプ | 実装コード（`gd` と異なる場合がある） |
| `gr` | 参照一覧を表示（Telescope） | 使われている全箇所 |
| `K` | ホバー情報を表示 | ドキュメント・型情報 |
| `Ctrl+o` | 前の場所へ戻る | ジャンプ後に元の位置へ戻る |
| `<Leader>ca` | コードアクション（クイックフィックス等） | |
| `<Leader>rn` | シンボルのリネーム | |
| `<Leader>cf` | フォーマット | |

> `gd` / `gi` は Telescope 経由で表示される。候補を選択すると自動でウィンドウが閉じてジャンプする。
> React コンポーネント（`<MyComponent />`）の上で `gd` → コンポーネントの定義ファイルへ。
> バレルファイル（index.ts）経由の場合は 2 回 `gd` が必要。
> `gd` でimportファイルや型定義に飛んだ場合は `gi` で実装コードへ、`Ctrl+o` で元の場所に戻る。

```
:LspRestart → LSP を再起動
```

## バッファ・ウィンドウ・ファイル操作

### 概念の整理

| 用語 | 意味 |
|------|------|
| バッファ | 開いているファイルのデータ（メモリ上）。上部タブバーに表示される |
| ウィンドウ | バッファを表示する画面の分割領域。1画面に複数持てる |
| タブ | ウィンドウのレイアウトをまとめたもの（通常は使わない） |

| `<Leader>e` | neo-tree を開いて選択する |

### ファイルを分割して開く

#### Telescope から分割で開く（推奨）

`<Leader>ff` などでファイルを選択中に以下のキーを押す:

| キー | 動作 |
|------|------|
| `Enter` | 現在のウィンドウで開く |
| `Ctrl+v` | 右に垂直分割して開く |
| `Ctrl+x` | 下に水平分割して開く |

#### コマンドで分割する

| コマンド | 動作 |
|---------|------|
| `:vsplit` / `:vsp` | 現在のファイルを右に垂直分割 |
| `:split` / `:sp` | 現在のファイルを下に水平分割 |
| `:vsp path/to/file` | 別ファイルを右に垂直分割で開く |
| `:sp path/to/file` | 別ファイルを下に水平分割で開く |

#### キーバインドで分割する

| キー | 動作 |
|------|------|
| `Ctrl+w v` | 現在のファイルを右に垂直分割 |
| `Ctrl+w s` | 現在のファイルを下に水平分割 |

#### 分割後に別ファイルを開く

```
1. Ctrl+w v で垂直分割
2. Ctrl+l で右ウィンドウに移動
3. <Leader>ff で別のファイルを検索して開く
```

| `<Leader>w=` | 全ウィンドウを均等サイズに揃える |
| `<Leader>wm` | 現在ウィンドウを最大化 |

### ファイルを閉じる

Neovim では「ファイルを閉じる」= バッファを削除する操作。ファイル自体は削除されない。

| キー / コマンド | 動作 |
|----------------|------|
| `<Leader>bd` | 現在のバッファを閉じる |
| `:bd` | バッファを削除（タブバーから消える） |
| `:q` | ウィンドウを閉じる（バッファはタブバーに残る） |
| `:wq` または `ZZ` | 保存して閉じる |
| `:q!` または `ZQ` | 保存せずに閉じる（変更を破棄） |

> `:q` と `:bd` の違い:
> `:q` はウィンドウを閉じるだけ（バッファはタブバーに残る）
> `:bd` はバッファも削除する（タブバーから消える）
>
> バッファが1つだけの状態で `:bd` すると Neovim が終了する。

## 閉じる・キャンセルする操作まとめ

操作後に「元に戻る」「キャンセルする」方法を状況別にまとめる。

### フロートターミナル（`<Leader>t`）

| 状態 | 操作 | 動作 |
|------|------|------|
| ターミナル入力中 | `<Leader>t` | トグルで隠す（プロセスは継続） |
| ターミナル入力中 | `Ctrl+\` → `Ctrl+n` | ノーマルモードへ切り替え |
| ノーマルモード | `:q` または `q` | フロートを閉じる（プロセスは継続） |
| ターミナル入力中 | `exit` または `Ctrl+d` | シェルを終了してフロートを閉じる |

> `<Leader>t` で隠しても中のプロセスは生きている。再度 `<Leader>t` で再表示できる。
> 完全に終了したい場合は `exit` でシェルを終了する。

### neo-tree（エクスプローラー）のフィルター検索

neo-tree 内で `/` を押すとファイル名のフィルタリングが始まる。

| 状態 | 操作 | 動作 |
|------|------|------|
| フィルター入力中 | `Esc` | フィルターをキャンセルして全ファイル表示に戻る |
| フィルター入力中 | `Backspace` | 入力文字を1文字削除 |
| フィルター適用後 | `<` | フィルタークリア（全表示に戻る） |
| neo-tree が開いた状態 | `<Leader>e` | neo-tree を閉じて編集画面に戻る |
| neo-tree が開いた状態 | `q` | neo-tree を閉じる |

### Telescope（ファジーファインダー）

| 状態 | 操作 | 動作 |
|------|------|------|
| 検索入力中 | `Esc` | Telescope を閉じて元の画面に戻る |
| 検索入力中 | `Ctrl+c` | Telescope を閉じる |
| リスト選択中 | `Esc` | Telescope を閉じる |

### lazygit（`<Leader>gg`）

| 状態 | 操作 | 動作 |
|------|------|------|
| lazygit 内 | `q` | lazygit を閉じて Neovim に戻る |
| lazygit 内 | `Esc` | 現在のパネル/モードをキャンセル |

### フロートウィンドウ全般（ドキュメント参照など）

| 状態 | 操作 | 動作 |
|------|------|------|
| フロートウィンドウ内 | `q` | フロートを閉じて元のバッファに戻る |
| フロートウィンドウ内 | `:q` | フロートを閉じる |

> `<Leader>?u` などで開いたドキュメントビューアも `q` で閉じる。

### which-key（キーバインドガイド）

| 状態 | 操作 | 動作 |
|------|------|------|
| which-key 表示中 | `Esc` | which-key を閉じて操作をキャンセル |
| which-key 表示中 | `Backspace` | 1階層上に戻る |

### Trouble（診断リスト）

| 状態 | 操作 | 動作 |
|------|------|------|
| Trouble が開いた状態 | `<Leader>d` | トグルで閉じる |
| Trouble が開いた状態 | `q` | Trouble を閉じる |

### 編集内容を取り消す（アンドゥ）

| キー | 動作 |
|------|------|
| `u` | 直前の変更を取り消す（アンドゥ） |
| `Ctrl+r` | 取り消しを戻す（リドゥ） |
| `:e!` | 最後に保存した状態にファイルをリセット（全変更破棄） |

## トラブルシューティング・リセット

### 症状別の対処

| 症状 | 対処 |
|------|------|
| neo-tree の幅がおかしい | `<Leader>w=` でウィンドウサイズをリセット |
| neo-tree が消えない | neo-tree 内で `q` |
| セッション復元後に不要なファイルが残る | `<Leader>qc` → `ZZ` → 再起動 |
| LSP が応答しない | `:LspRestart` |
| 全部おかしい | `ZZ` で終了して再起動 |

### neo-tree の幅・レイアウトがおかしい時

| キー | 動作 |
|------|------|
| `<Leader>w=` | 全ウィンドウを均等サイズにリセット |
| `q`（neo-tree 内）→ `<Leader>e` | neo-tree を閉じて開き直す |
| `<Leader>qc` | セッションファイルをクリア（次回起動時にクリーンな状態で立ち上がる） |

### Neovim のリロード・再起動

設定ファイルだけ再読み込み（プラグインは反映されないことが多い）:

```
:source ~/.config/nvim/init.lua
```

Neovim を終了して再起動（最も確実）:

| キー / コマンド | 動作 |
|----------------|------|
| `ZZ` | 全バッファを保存して終了 |
| `:qa` | 保存せず終了（変更があれば警告） |
| `:qa!` | 強制終了（保存しない） |

終了後、ターミナルで `nvim` を実行するとセッションが自動復元される。

### セッション操作

| キー | 動作 |
|------|------|
| `<Leader>qs` | セッション復元（現在ディレクトリ） |
| `<Leader>ql` | 最後のセッションを復元 |
| `<Leader>qd` | セッション保存を停止（このセッション） |
| `<Leader>qc` | セッションファイルをクリア（壊れた時のリセット用） |

## 注意事項

### Ctrl+a について

`Ctrl+a` は tmux prefix に取られているため Neovim に届かない。
数値のインクリメントには `+` キーを使う。

### tmux との使い分け

| 操作 | 推奨キー |
|------|---------|
| ペイン分割（横） | `Cmd+n`（Ghostty → tmux） |
| ペイン分割（縦） | `Cmd+Shift+n`（Ghostty → tmux） |
| Neovim 内の分割 | `Ctrl+w v`（縦）/ `Ctrl+w s`（横） |
| ペイン/ウィンドウ移動 | `Ctrl+h/j/k/l`（Neovim/tmux 統合） |
| tmux window の切替 | `Cmd+1〜9` または `Ctrl+Tab` |

## 開発ワークフロー

### worktree 作成から実装まで

worktree を作成すると以下のレイアウトが自動で立ち上がる:

```
Window 1 "terminal"
  pane 1.1 — 汎用ターミナル（Claude Code を起動する場所）
  pane 1.2 — npm run watch（フロントサーバー）

Window 2 "nvim"
  Neovim がワークツリールートで起動済み
```

典型的な操作の流れ:

```
1. prefix + w / p で worktree を選択または作成
2. Window 2 (nvim) に自動フォーカス → Neovim が開いている
3. <Leader>ff でファイルを開いて実装開始

6. prefix + Ctrl+g で Neovim からプロンプトを書いて Claude Code に送信
```

## ターミナルの開き方

### 方法 1: Neovim 内フロートターミナル（snacks.nvim）

| キー | 動作 |
|------|------|
| `<Leader>t` | フロートターミナルをトグル |

- Neovim の画面上にフロートウィンドウとしてターミナルが開く
- `Ctrl+\` → `Ctrl+n` でターミナルのノーマルモードへ（コピーなど）
- `i` で入力モードに戻る
- ターミナル内で `exit` または `Ctrl+d` でフロートを閉じる

### 方法 2: tmux ペインに分割して開く

Neovim を閉じずに隣のペインでターミナルを使いたい場合:

| キー | 動作 |
|------|------|
| `Cmd+n` | Neovim の右に新しいペインを作成（横分割） |
| `Cmd+Shift+n` | Neovim の下に新しいペインを作成（縦分割） |
| `Ctrl+h/j/k/l` | Neovim ↔ ターミナルペインをシームレスに移動 |

### 方法 3: Neovim 内の垂直・水平分割

Neovim のウィンドウとして別コマンドを実行したい場合:

```
:terminal         — 現在のウィンドウでターミナルを開く
:split | terminal — 水平分割でターミナルを開く
:vsplit | terminal — 垂直分割でターミナルを開く
```

## Claude Code ターミナルへの移動

### worktree のペイン構成での典型操作

worktree 作成直後のレイアウト:

```
Window 1 (terminal)         Window 2 (nvim)
┌──────────┬──────────┐     ┌─────────────────┐
│ pane 1.1 │ pane 1.2 │     │   Neovim        │
│ (shell)  │ (watch)  │     │                 │
└──────────┴──────────┘     └─────────────────┘
```

**Neovim → Claude Code のターミナルへ移動する手順**:

```
方法 A: tmux window 切替（推奨）
  Cmd+1  →  Window 1 の pane 1.1（Claude Code が動いているペイン）へ

方法 B: ペイン間を Ctrl+h/j/k/l で移動
  Neovim と Claude Code が同じ window 内の隣接ペインにある場合のみ使える

方法 C: prefix + Ctrl+g（Neovim からプロンプトを送信）
  window を切り替えずに、Neovim で書いたテキストを Claude Code に直接送れる
  → NeovimからClaude Codeに移動せず作業が完結する（最推奨）
```

**Claude Code → Neovim に戻る手順**:

```
Cmd+2  →  Window 2 (nvim) へ戻る
```

### prefix + Ctrl+g の使い方（Neovim でプロンプト編集）

1. 任意のペインで `prefix + Ctrl+g` を押す
2. フロートポップアップで Neovim が開く（空の markdown ファイル）
3. プロンプトを書く（複数行も可）
4. `:wq` で保存・送信 / `:q!` でキャンセル
5. 自動的に Claude Code が動いているペインにテキストが送信される

## 修正内容の確認（Git 管理）

### inline 確認（gitsigns）

コーディング中にガター（行番号左）で変更行を即確認:

| 表示 | 意味 |
|------|------|
| `│`（緑） | 追加した行 |
| `│`（黄） | 変更した行 |
| `_` / `‾` | 削除した行 |

| キー | 動作 |
|------|------|
| `]h / [h` | 次/前の変更ハンクへ移動 |
| `<Leader>gp` | ハンクの diff をプレビュー（変更前後を確認） |
| `<Leader>gd` | ファイル全体の diff を分割表示 |
| `<Leader>gb` | 現在行の git blame（誰がいつ書いたか） |

### 差分ビュー（diffview）

複数ファイルの差分をファイルツリー付きで一覧したい場合に使う。

| キー | 動作 |
|------|------|
| `<Leader>gv` | 差分ビューを開く（全変更ファイル一覧 + diff） |
| `<Leader>gh` | 現在ファイルのコミット履歴を開く |
| `<Leader>gH` | リポジトリ全体のコミット履歴を開く |
| `q` | diffview を閉じる |

diffview は「ファイルパネル（左）」と「diff ビュー（右）」の 2 ペインで構成される。

ファイルパネル（左）— ステージング操作:

| キー | 動作 |
|------|------|
| `j / k` | ファイルを上下に移動 |
| `Enter` / `l` | 選択ファイルの diff を表示 |
| `s` または `-` | 選択ファイルをステージ / アンステージ |
| `S` | 全ファイルをステージ |
| `U` | 全ファイルをアンステージ |
| `X` | 選択ファイルの変更を破棄 |
| `Tab` / `Shift+Tab` | 次/前のファイルへ移動 |
| `R` | ファイル一覧を更新 |

diff ビュー（右）— 差分確認:

| キー | 動作 |
|------|------|
| `]c / [c` | 次/前の変更ハンクへジャンプ（Vim 標準） |
| `Tab` / `Shift+Tab` | 次/前のファイルへ移動 |
| `<Leader>e` | ファイルパネルにフォーカスを移す |
| `<Leader>b` | ファイルパネルの表示をトグル |
| `g?` | キーバインド一覧を表示 |

### まとめて確認・操作（lazygit）

| キー | 動作 |
|------|------|
| `<Leader>gg` | lazygit を開く（全変更をまとめて確認・操作） |
| `<Leader>gl` | リポジトリ全体のコミットログ |
| `<Leader>gL` | 現在ファイルのコミットログ |

lazygit 内での操作:

| キー | 動作 |
|------|------|
| `Space` | ファイルをステージ/アンステージ |
| `c` | コミット |
| `P` | Push |
| `p` | Pull |
| `d` | 選択ファイルの diff を確認 |
| `q` | lazygit を閉じて Neovim に戻る |
